# Quick Workflow

**Fewer personas. Faster cycle. Skips deep research.**

---

## When to Use

Use the Quick workflow when:

- [PLACEHOLDER — e.g., You already have enough context on the subject that deep research would surface nothing new]
- [PLACEHOLDER — e.g., Speed matters — you need output today, not tomorrow]
- [PLACEHOLDER — e.g., The project is medium-complexity — not trivial (use Simple), not high-stakes (use Full)]
- [PLACEHOLDER — e.g., You have run Full on this type of project before and know the Researcher persona consistently produces similar findings]

**Typical use cases:**
- [PLACEHOLDER — e.g., Warm lead outreach (you already know the company and context from a previous conversation)]
- [PLACEHOLDER — e.g., Medium-length content (1,000–2,000 words) where structure matters more than deep research]
- [PLACEHOLDER — e.g., Repeat project types where patterns are established and learnings are in PROJECT-LEARNINGS.md]

**Skipped personas:**
- [PLACEHOLDER — e.g., 02 — Researcher] — [PLACEHOLDER — e.g., Skipped because: existing context is sufficient / research would take longer than the value it adds / subject is opinion-based and doesn't require external evidence]

> If the Critic scores below 75 on Quick, default to Full on the next similar project. The Researcher's absence is usually the root cause.

```
[PHASE r — Growth/Exploration]
        │
        ▼

Mandatory human checkpoint: YES
        │
        ▼
[02 — PLACEHOLDER (e.g., Researcher)] ← SKIPPED

        ▼

[PHASE K — Conservation/Production]
        │
        ▼
[04 — PLACEHOLDER (e.g., Creator)]
Executes strategy brief.
Human checkpoint: OPTIONAL
        │
        ▼
[05 — PLACEHOLDER (e.g., Editor / Validator)]
Enforces constitution. Mechanical checks.
Human checkpoint: NONE

[PHASE Ω — Release/Quality Gate]
        │
        ▼

Scores against 5 dimensions. Issues verdict.

        │

        ├─ REVISE (70–84) → Return to [04]
        ├─ RESTRUCTURE (50–69) → Consider upgrading to Full
        └─ RELEASE (<50) → α-phase + upgrade to Full next time
```

## Upgrade Path

**Upgrade to Full when:**
- Critic scores below 75 on Personalisation or Strategic Alignment dimensions
- You find yourself editing the output heavily because it feels generic
- The subject is unfamiliar and research would reduce guesswork

**How to upgrade mid-project:**
1. Stop after Qualifier output
2. Run Researcher before proceeding to Strategist
3. Update CONTEXT.md to note the upgrade and why
4. Append to MEMORY.md: "Upgraded Quick → Full. Reason: [specific reason]."
