# Full Workflow

**All personas. Highest quality. Longest cycle.**

---

## When to Use

Use the Full workflow when:

- [PLACEHOLDER — e.g., The lead / subject / project warrants deep research before output (high-value opportunity, complex subject, strategic importance)]
- [PLACEHOLDER — e.g., Quality matters more than speed — you have time to run every persona and human review is possible at key checkpoints]
- [PLACEHOLDER — e.g., This is a new type of project and you want to establish a quality baseline]
- [PLACEHOLDER — e.g., Previous Quick or Simple attempts scored below 80 — the Full workflow's additional research persona often surfaces the insights that change the strategic angle]

**Typical use cases:**
- [PLACEHOLDER — e.g., Enterprise lead outreach (high-ACV deals where one good email is worth several hours of research)]
- [PLACEHOLDER — e.g., Long-form content (2,000+ words where structural integrity and research depth determine quality)]
- [PLACEHOLDER — e.g., Strategic documents where getting it wrong has real consequences]

## Phase Flow

```
[PHASE r — Growth/Exploration]
        │
        ▼
[01 — PLACEHOLDER (e.g., Qualifier)]
Defines what we're working with. Confirms this is worth pursuing.
Mandatory human checkpoint: YES
        │
        ▼
[02 — PLACEHOLDER (e.g., Researcher)]
Deep context. Surfaces signals, trigger events, evidence.
Human checkpoint: OPTIONAL
        │
        ▼
[03 — PLACEHOLDER (e.g., Strategist)]
Resolves all ambiguity. Produces a brief the Creator can execute without questions.
Human checkpoint: OPTIONAL (recommended for new project types)

[PHASE K — Conservation/Production]
        │
        ▼
[04 — PLACEHOLDER (e.g., Copywriter / Writer / Builder)]
Produces the primary output artifact. Executes the strategy brief.
Human checkpoint: OPTIONAL
        │
        ▼
[05 — PLACEHOLDER (e.g., Editor / Validator)]
Enforces constitution. No Kill List violations. Structural checks.
Human checkpoint: NONE (mechanical validation — auto-passes or sends back)

[PHASE Ω — Release/Quality Gate]
        │
        ▼
[06 — PLACEHOLDER (e.g., Critic)]
Scores against 5 dimensions. Issues final verdict.
Mandatory human checkpoint: YES (before delivery)
        │
        ├─ APPROVE (≥85) → Human reviews → Ship
        ├─ REVISE (70–84) → Return to [04]
        ├─ RESTRUCTURE (50–69) → Return to [03]
        └─ RELEASE (<50) → α-phase triggers
                │
                ▼
        [PHASE α — Evolution (conditional)]
                │
                ▼
        [07 — PLACEHOLDER (e.g., Evolutionist)]
        Root cause analysis. Constitution + learnings update.
        Returns to r-phase for next project.
        Mandatory human checkpoint: YES (constitution changes require approval)
```

**Purpose:** Gather all available information. Resolve unknowns. Produce a foundation the K-phase can build on without backtracking.

| **Output** | [PLACEHOLDER — e.g., Lead profile with ICP score, company context, role context, trigger event hypothesis] |
| **Human checkpoint** | REQUIRED — [PLACEHOLDER — e.g., Confirm the lead is worth pursuing before committing research time] |
| **Advance criteria** | [PLACEHOLDER — e.g., ICP score documented. Company, role, and at least one trigger hypothesis identified.] |

| **Human checkpoint** | OPTIONAL — [PLACEHOLDER — e.g., Review if research surfaces unexpected findings that change the strategic direction] |
| **Advance criteria** | [PLACEHOLDER — e.g., At least one strong trigger event identified. Pain point hypothesis confirmed or revised with evidence.] |

| **Output** | [PLACEHOLDER — e.g., Validation report with PASS/FAIL verdict and specific issues found] |

### Phase Ω — Release/Quality Gate

**Purpose:** Independent quality assessment. Issue a final verdict. Do not soften scores.

| Persona | [PLACEHOLDER — e.g., 06 — Critic] |
|---------|-------------------------------------|
| **Purpose** | [PLACEHOLDER — e.g., Scores the output against five quality dimensions. Issues one of four verdicts: APPROVE, REVISE, RESTRUCTURE, RELEASE. The Critic is adversarial by design — their job is to find problems, not confirm quality.] |
| **Output** | [PLACEHOLDER — e.g., Critique report with per-dimension scores, total score, verdict, and specific revision instructions] |
| **Human checkpoint** | REQUIRED — [PLACEHOLDER — e.g., Human reviews critique report and final output before delivery/publication. Human can override Critic verdict with documented reason.] |
| **Advance criteria** | [PLACEHOLDER — e.g., Critique report complete with scores for all five dimensions. Verdict issued. Human has reviewed and confirmed.] |

### Phase α — Evolution (Conditional)

**Triggered by:** RELEASE verdict (score <50) only.

**Purpose:** Root cause analysis. Update the system so the same failure does not recur.

| Persona | [PLACEHOLDER — e.g., 07 — Evolutionist] |
|---------|------------------------------------------|
| **Purpose** | [PLACEHOLDER — e.g., Analyses why the project failed. Identifies the root cause (was it the strategy? the execution? the brief? the constitution?). Proposes specific, evidence-based updates to the constitution and project learnings.] |
| **Output** | [PLACEHOLDER — e.g., Root cause analysis report + proposed constitution changes + learnings entries to add to PROJECT-LEARNINGS.md] |
| **Human checkpoint** | REQUIRED — [PLACEHOLDER — e.g., All constitution changes require human approval before being applied. Human reviews the root cause analysis and confirms or modifies proposed changes.] |
| **Advance criteria** | [PLACEHOLDER — e.g., Root cause identified with evidence. Constitution change proposals reviewed and approved (or rejected) by human. PROJECT-LEARNINGS.md updated.] |

## Upgrade / Downgrade Paths

**If you're on Quick or Simple and it's not working:**
→ Upgrade to Full. The Researcher persona is usually what's missing when Quick/Simple outputs score poorly on personalisation or strategic alignment dimensions.

**If Full is taking too long:**
→ Consider Quick (skip Researcher). Only downgrade if you have strong existing context for the subject — enough that a Researcher would surface nothing new.

**Never downgrade to Simple if:**
- [PLACEHOLDER — e.g., The opportunity is high-value — quality matters more than speed]
- [PLACEHOLDER — e.g., The subject is complex — Simple's minimal pipeline produces generic output on complex subjects]
